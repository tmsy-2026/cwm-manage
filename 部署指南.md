# 幕墙管理系统 v3.0 部署指南

## 方案一：Netlify 部署（永久免费，互联网访问）

### 第一步：配置 Firebase（10分钟）

1. 访问 https://console.firebase.google.com
2. 点击「添加项目」→「幕墙管理系统」
3. 创建后进入「构建」→「Firestore Database」→「创建数据库」→ 选择「测试模式」→「亚洲-east1」
4. 进入「构建」→「Authentication」→「开始使用」→ 启用「电子邮件/密码」
5. 进入「项目设置」→ 滚动到「你的应用」→ 点击 web 图标 `<>` → 注册应用 → 复制 Firebase SDK 配置

### 第二步：填入配置

打开 `firebase-config.js`，粘贴 Firebase 控制台给你的配置代码：
```javascript
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project-id",
  ...
};
```

### 第三步：部署到 Netlify（3分钟）

1. 访问 https://app.netlify.com/drop
2. **直接把整个 `f:\workbuddy测试01-幕墙工程管理系统` 文件夹拖进去**
3. Netlify 会自动分配一个网址，如 `random-name-123.netlify.app`
4. 团队成员直接用这个网址访问即可

### 第四步：让同事加入

1. 把网址发给同事：`https://你的名字.netlify.app`
2. 同事用自己的手机/电脑浏览器打开
3. 输入账号密码登录（和之前一样：admin/admin123 等）
4. **点击手机浏览器的「添加到主屏幕」**，就变成一个 App 图标了

## 方案二：内网访问（无需网络，仅限同一WiFi）

### 在你电脑上运行服务器

```powershell
cd f:\workbuddy测试01-幕墙工程管理系统
python -m http.server 7788 --bind 0.0.0.0
```

### 同事手机访问

确保同事手机和你的电脑在**同一WiFi网络**，然后在手机浏览器输入：
```
http://192.168.1.12:7788
```
（同局域网内有效，关机后无法访问）

## 数据说明

- **Firebase 模式**：所有数据存储在 Google Firebase 云端，团队成员实时共享
- **离线支持**：Firebase 支持离线缓存，网络恢复后自动同步
- **免费额度**：每月 50GB 读取/写入，小团队完全够用
- **导出数据**：系统设置 → 数据管理 → 导出数据，可导出 JSON 备份
